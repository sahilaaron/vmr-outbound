Local Tools local development only

Safe controls for exercising the workbench with synthetic data. These tools refuse to run unless the app environment is local and the database host is a loopback address — a live or remote database (including RDS) can never be touched.

Load fixtures

Synthetic data only — no real people or companies

Reset local data

Destructive — local database only

Clearing removes all campaigns, contacts, imports, suppressions, and audit history from the local database (the schema and migration state survive). A fresh audit event records the reset. Type RESET to confirm.

Reset to a known demo state = clear everything, then load both fixtures.

Guard rails: these actions call ensure_local_database(), which raises unless APP_ENV=local and the configured database host is 127.0.0.1 / localhost / ::1. They are absent from the navigation and return 404 outside local development. Fixture loads and resets write audit events, so the local audit trail shows what happened.
